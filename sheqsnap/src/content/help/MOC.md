# Management of Change (MOC)

## What is MOC?

Management of Change (MOC) is a formal process used to evaluate and control changes to processes, equipment, documents, personnel, and procedures before they are implemented. It ensures that changes are properly assessed for safety, compliance, and operational risks before taking effect.

## When to Raise a Change Request

A change request should be raised whenever there is a proposed change to:
- **Process** — changes to how work is performed or production processes
- **Equipment / Plant** — modifications, upgrades, or replacements of machinery or plant
- **Document / Procedure** — revisions to safety procedures, work instructions, or policies
- **Personnel** — changes to key roles, responsibilities, or organisational structure
- **Temporary Change** — short-term deviations from standard operating procedures
- **Other** — any other change that may have safety, environmental, or compliance implications

When in doubt, raise a change request. It is better to document and assess a minor change than to proceed without review.

## The Approval Workflow

Change requests follow a structured approval process:

**Draft** → **Pending Approval** → **Approved** → **Implemented** → **Closed**

| Stage | Description |
|-------|-------------|
| Draft | The change request has been created but not yet submitted for review |
| Pending Approval | The request has been submitted and is awaiting review by a Manager or Admin |
| Approved | The change has been reviewed and approved; it may now proceed to implementation |
| Implemented | The change has been carried out; an implementation date is recorded |
| Closed | The change has been reviewed post-implementation and formally closed |
| Rejected | The change request was not approved; a rejection reason is recorded |

## Change Types

Select the most appropriate change type when raising a request. This helps with filtering, reporting, and ensures the correct people review the request.

## Why MOC Matters for SHEQ Compliance

Uncontrolled changes are one of the leading causes of workplace incidents. A formal MOC process:
- Ensures all risks associated with a change are identified and mitigated before implementation
- Creates an auditable record of changes and approvals
- Demonstrates due diligence to regulators and auditors
- Prevents "change creep" — where small, undocumented changes accumulate and create systemic risk
- Supports incident investigation by providing a record of recent changes

## Risk Assessment

When raising a change request, include a risk assessment describing potential hazards and mitigations. This does not replace a formal risk assessment process but provides a summary of known risks to inform the approver's decision.
